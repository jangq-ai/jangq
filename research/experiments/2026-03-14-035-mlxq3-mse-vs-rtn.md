## Analysis

### MSE-optimal vs RTN
- RTN top token: 'What' (repeating prompt)
- MSE top token: 'assistant' (chat template artifact)
- MSE is slightly more coherent but still wrong

### Why MSE doesn't fully fix it

MSE-optimal clipping (searching 15 ratios from 0.7 to 1.0) helps by
trimming outlier values, but the fundamental issue remains: 3-bit MLP
at block_size=64 only has 8 quantization levels per block.

The step size Δ = range/7 for each block. For MLP weights with typical
range [-0.5, 0.5], Δ ≈ 0.14. That's a 14% granularity — too coarse
for preserving the fine weight structure.

### Speed observations

| Model | Decode Speed |
|-------|-------------|
| MLXQ 8-bit (release) | 28.3 tok/s |
| MLXQ-3 MSE (release) | 24.9 tok/s |
| MLXQ-3 RTN (debug) | ~15 tok/s |

3-bit is slightly slower than 8-bit (24.9 vs 28.3) despite being smaller.
This is because our GEMV kernel has branch divergence when processing
mixed bit-width blocks (some 3-bit, some 6-bit).

## What's Left to Try

1. **AWQ + MSE + profile combined** — should stack the improvements
2. **Group_size=32** instead of 64 — smaller groups = more scales =
   effectively more bits, but more overhead
3. **NormalFloat quantization** — non-uniform levels optimized for
   Gaussian distributions
4. **Test on 7B+ models** — where 3-bit actually works in practice

## Key Insight

The 3B model at 3-bit is at the edge of what's possible without
advanced quantization (GPTQ/NF). For the paper, we should focus on:
1. 7B+ models where 3-bit works much better
2. The proven MLP=3/attn=6 strategy (experiment 028)
3. The theoretical framework (rate-distortion, AWQ math)
