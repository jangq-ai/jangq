## Model

- **MiniMax-M2.5**: 230B total params, 10B active per token
- 256 experts, 8 active per token
- 62 layers, GQA (48 heads / 8 KV heads)
- Hidden: 3072, intermediate: 1536
- Source format: FP8 E4M3 with block scale_inv (128×128 blocks)
- Source size: 214 GB (FP8 safetensors, 126 shards)
- Location: Mac Studio `/Volumes/EricsLLMDrive/MiniMax-M2.5/`

## Architecture Details

MiniMax-M2.5 uses:
- Standard GQA attention (q/k/v/o_proj) — NOT hybrid like Qwen3.5
- MoE with `block_sparse_moe` naming (like Mixtral)
- Expert weights: per-expert 2D tensors (`experts.N.w1.weight`, `experts.N.w3.weight`)
- No `w2` — uses fused gate_proj/up_proj naming after sanitize
- `e_score_correction_bias` — expert load balancing bias
- `q_norm`/`k_norm` — QK-norm (1D, passes through as fp16)

## FP8 Handling

MiniMax weights are stored as FP8 E4M3 (float8_e4m3fn) with companion `weight_scale_inv` tensors.

Dequantization formula:
```python
fp8_float = fp8_e4m3_to_float32(raw_uint8)  # E4M3: 1 sign, 4 exp (bias=7), 3 mantissa
scale_inv = load_f32([rows/128, cols/128])    # block-wise inverse scale
weight = fp8_float * repeat_block(scale_inv, 128, 128)
```

JANG handles this via `jang_tools/fp8.py`:
- `fp8_e4m3_to_float32()` — vectorized E4M3 → float32 conversion
- `load_fp8_tensor()` — loads raw bytes, converts, applies block scales
- `is_fp8_model()` — detects FP8 safetensors
- `_scale_inv` tensors are skipped in calibrate.py and convert.py

## Conversion

- Profile: JANG_2S (CRITICAL=6, IMPORTANT=4, COMPRESS=2)
- Running on Mac Studio (192 GB RAM)
- Calibration: ~60 min (126 shards × 30 sec/shard, FP8 dequant per tensor)
- Quantization: TBD (vectorized packing should handle 15k+ tensors)
- **COMPLETED**: 2.06 avg bits, 54.96 GB qweight, 88 GB total on disk
- Conversion time: ~3.5 hours (53 min calibration + ~2.5 hrs quantization + write)
- 47,928 tensors quantized across 62 layers × 256 experts

## Architecture Detection Fixes Applied

1. `attn_type_list` guard — MiniMax has this field but is NOT deltanet (fixed)
2. `weight_scale_inv` skip — FP8 scale tensors correctly excluded from quantization
3. MoE detection — `num_local_experts: 256` correctly routes to MoE branch
